# Developer Account Management

Manage your Norai developer account, API keys, projects, and track your API usage.

## Create a Developer Account

Sign up for a Norai developer account to get started:

<ApiBlock
  method="POST"
  endpoint="/v1/developer/account"
  curl={`curl -X POST "https://api.norai.com/v1/developer/account" \\
  -H "Content-Type: application/json" \\
  -d '{
    "company_name": "Example Corp",
    "website": "https://example.com",
    "email": "dev@example.com"
  }'`}
  nodejs={`const response = await fetch('https://api.norai.com/v1/developer/account', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    company_name: "Example Corp",
    website: "https://example.com",
    email: "dev@example.com"
  })
});

const account = await response.json();`}
/>

### Example Response

```json
{
  "success": true,
  "developer": {
    "id": "dev_123",
    "company_name": "Example Corp",
    "website": "https://example.com",
    "email": "dev@example.com",
    "tier": "free",
    "created_at": "2025-12-03T10:30:00Z"
  },
  "project": {
    "id": "proj_abc123",
    "name": "Default Project",
    "api_key": "nk_your_api_key_here"
  }
}
```

## Get Developer Account

Retrieve your developer account details:

<ApiBlock
  method="GET"
  endpoint="/v1/developer/account"
  curl={`curl -X GET "https://api.norai.com/v1/developer/account" \\
  -H "Authorization: Bearer YOUR_API_KEY"`}
  nodejs={`const response = await fetch('https://api.norai.com/v1/developer/account', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const account = await response.json();`}
/>

## API Keys Management

### List API Keys

Get all API keys for your account:

<ApiBlock
  method="GET"
  endpoint="/v1/developer/credentials"
  curl={`curl -X GET "https://api.norai.com/v1/developer/credentials" \\
  -H "Authorization: Bearer YOUR_API_KEY"`}
/>

### Example Response

```json
{
  "success": true,
  "keys": [
    {
      "id": "key_1",
      "name": "Production Key",
      "key": "nk_abcdefghijklmnopqrstuvwxyz",
      "project_id": "proj_abc123",
      "permissions": ["events:write", "recommendations:read"],
      "created_at": "2025-12-01T10:30:00Z",
      "last_used": "2025-12-03T14:25:00Z",
      "status": "active"
    }
  ]
}
```

### Create API Key

Generate a new API key:

<ApiBlock
  method="POST"
  endpoint="/v1/developer/credentials"
  curl={`curl -X POST "https://api.norai.com/v1/developer/credentials" \\
  -H "Authorization: Bearer YOUR_API_KEY" \\
  -H "Content-Type: application/json" \\
  -d '{
    "name": "Production Key",
    "project_id": "proj_abc123",
    "permissions": ["events:write", "recommendations:read", "segments:read"],
    "expires_at": "2026-12-31T23:59:59Z"
  }'`}
/>

### Example Response

```json
{
  "success": true,
  "key": {
    "id": "key_2",
    "name": "Production Key",
    "key": "nk_newapikeystring",
    "permissions": ["events:write", "recommendations:read", "segments:read"],
    "expires_at": "2026-12-31T23:59:59Z",
    "created_at": "2025-12-03T10:30:00Z"
  }
}
```

**Important**: The API key is only shown once. Store it securely.

### Revoke API Key

Revoke an API key:

<ApiBlock
  method="DELETE"
  endpoint="/v1/developer/credentials/:keyId"
  curl={`curl -X DELETE "https://api.norai.com/v1/developer/credentials/key_2" \\
  -H "Authorization: Bearer YOUR_API_KEY"`}
/>

## Account Tiers

Norai offers different account tiers:

### Free Tier
- 10,000 events/month
- 1,000 API calls/month
- 1 project
- Community support

### Professional Tier
- 1,000,000 events/month
- 100,000 API calls/month
- 5 projects
- Email support
- Advanced analytics

### Enterprise Tier
- Unlimited events
- Unlimited API calls
- Unlimited projects
- Dedicated support
- Custom SLA
- On-premise deployment options

## Best Practices

1. **Separate environments**: Use different API keys for development, staging, and production
2. **Monitor usage**: Check usage statistics regularly to optimize costs
3. **Rotate keys**: Periodically rotate API keys for security
4. **Set expiration**: Always set expiration dates on API keys
5. **Minimum permissions**: Create keys with only the permissions they need
